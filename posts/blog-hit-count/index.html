블로그 히트 카운트 읽기

간단한 웹 데이터는 읽는 방법도 간단하다. 웹 페이지 주소와 데이터를 지정할 수 있는 CSS selector만 있으면 된다. 블로그에 글을 쓰고 있는데 블로그 통계 정보를 알고 싶어서 아래 명령으로 확인하고 있다. 두 가지 명령을 연결해서 사용한다. Curl로 웹 페이지를 읽어오고 hgrep으로 대상 HTML 웹 페이지 파일에서 CSS selctor를 사용하여 텍스트를 찾는다. 현재 2000 카운트를 넘었다. 약 3년된 블로그니 인기는 없는 편이다^^.

$ curl -s https://agvim.wordpress.com/ | hgrep -t '#blog-stats-18 > ul > li'
2,023 hits

비슷한 방법으로 서울의 현재 기온을 터미널에서 읽을 수도 있다.

$ curl -s https://www.accuweather.com/ko/kr/seoul/226081/weather-forecast/226081 \
| hgrep -t "#current-city-tab > a > span.local-temp"
28°C

또 다른 예로, 이 블로그는 Metalsmith라는 Static Site Generator를 사용하여 작성되었다. SSG는 StaticGen에서 여럿 찾을 수 있는데 각자 자신에 대해서 한 줄 설명을 하고 있다. 그 내용을 한 단어로 요약하고 싶어서 엄청 단순한 방법을 사용했다. 한 줄 설명을 모두 모아서 단어로 구분하고 가장 많이 언급된 단어를 찾기로 했다. 위에서 사용한 명령어를 조금 더 응용하면 조금 길지만 결과는 간단한 아래 명령을 만들수 있다.

$ curl -s https://www.staticgen.com/ \
| hgrep -t "div.description" \
| sed -e 's/\s\+/\n/g' \
| sed -e 's/\.//g' \
| sed -e 's/,//g' \
| tr [:upper:] [:lower:] \
| sort \
| uniq -c \
| sort -r \
| head -10
  139 static
  112 generator
  108 a
   78 site
   62 and
   46 for
   40 is
   39 in
   34 simple
   31 written

Curl로 시작하는 명령어는 10줄이나 되지만 이해하기 어렵지 않다. 이 명령어를 수행하면 가장 많이 언급된 10개의 단어를 볼 수 있다. 어설픈 결론을 지으면 툴 제목인 Static Site Generator를 제외하고 simple 단어가 가장 많이 언급되었으니 이 툴들은 자신의 간단함을 자랑한다는 것을 알 수 있다. 실제로 그렇다는 것이 아니니 과정만 즐기길 바란다. 스크랩 혹은 크롤링등의 고급적인 방법이 아니더라고 고급 프로그래밍 언어를 사용하지 않더라고 작은 명령어 조함으로 웹 데이터를 가공할 수 있다. 더구나 한번 만들어 놓으면 자주 사용할 수 있으며 시간순으로 저장하면 이력이 된다. 개인 툴 박스에 넣어두기에 딱이다.

명령어설명
curl -s https://www.staticgen.com/웹 페이지를 읽는다.
hgrep -t “div.description”한 줄 설명을 지정한다.
sed -e ‘s/\s+/\n/g’공백을 행 변경 문자로 바꾼다.
sed -e ‘s/.//g’. (dot)를 지운다.
sed -e ‘s/,//g’, (comma)를 지운다.
tr [:upper:] [:lower:]모두 소문자로 바꾼다.
sort정렬한다.
uniq -c단어 앞에 반복된 총 회수를 추가한다.
sort -r역순으로 정렬한다.
head -10처음 10줄 만 보여 준다.
