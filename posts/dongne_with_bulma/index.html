동네 도서관에서 책 찾기
2018/02/25


동네 도서관에서 책 찾기 웹 서비스를 만들어 사용하고 있었는데 불편해서 수정했다. 그 과정에 무엇이 불편했고 어떻게 수정했는지 정리했다.

불편한 점

책 이름을 입력한 후 나타나는 회색 바탕의 도서관 이름들의 의미가 분명하지 않다. 검색 시작 버튼을 눌렀는데 내용에 비해 화면이 너무 작아서 검색 결과를 볼 수 없다. 다시 시작 버튼의 의미가 분명하지 않다. 간단한 기능임에도 어떻게 사용해야 하는지 자꾸 궁금해지는 서비스다.

아래 입력란에 책 제목을 입력해서 테스트할 수 있다. 아래 서비스는 늦게 로딩되는 경우가 있는데 그 때는 10초 정도 걸린다. 무료 서버를 사용하고 있기 때문이다.

시나리오

어떤 순서로 동작할지 시나리오를 정리했다.

  1. 도서관을 선택한다.
  2. 책 제목의 일부를 입력한다.

더욱 단순한 UI

CSS 프레임워크로 이전에 사용하던 Bootstrap 대신 Bulma를 사용하기로 결정했다. 또한, UI를 새롭게 만들기 위해 도서관 검색 서비스에서 웹 API 서비스를 따로 분리했다. 이것을 사용하여 검색 UI를 다시 만들 생각이다. 사용자에게 보여줄 영역은 세 가지다

  • 서비스 내용을 알려줄 제목
  • 책 제목을 입력할 입력란
  • 검색된 책 제목 리스트

이 UI를 웹 API 서버인 dlserver 올렸다. public 디렉토리 제공 기능을 추가했다. 아래에서 바로 검색할 수 있다.

자바스크립트 프레임워크

자바스크립트 프레임워크는 종류가 많아서 선택이 중요하다. 그렇지만 시간이 부족함으로 그 동안 사용해 왔던 AngularJS를 사용할텐데 이전과 다르게 yo의 제너레이터를 사용하지 않고 하나의 js 파일을 읽어들이는 방법으로 사용했다.

public$ tree
.
├── app.js
└── index.html

App.js 파일 안에 AngularJS로 만드는 모든 자바스크립트 파일을 넣었다. 하나의 service와 하나의 controller를 사용했다. 아래 두 개의 인터페이스를 사용한다.

  • https://dlserver.herokuapp.com/libraryList
  • https://dlserver.herokuapp.com/?title=javascript&libraryName=판교

마무리

전체 코드를 다운로드하여 살펴보거나 온라인에서 기능을 확인할 수 있다. 온라인 서버는 무료 서버에서 동작함으로 잠들어 있다면 10초 정도 로딩 시간이 걸릴 수 있다. 최소한의 기능을 만들었다. 기능을 업데이트할 예정이다.
